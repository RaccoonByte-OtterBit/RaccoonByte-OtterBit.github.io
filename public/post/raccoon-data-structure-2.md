## 👀 변수가 자료구조라고??

일반적으로 자료구조를 언급할 때에 변수를 생각하고 말하지는 않습니다.
그렇지만 분명 변수도 자료구조의 하나라고 할 수 있습니다. 왜냐면 변수도 자료구조의 특성을 가지고 있기 때문이죠!

[이전 포스팅]()에서 자료구조는 "데이터를 담는 그릇"이라고 했습니다.
분명히 변수도 데이터를 담고 있기 때문에 자료구조입니다.
그저 단순한 형태이기 때문에 문제해결을 위한 자료구조로써 고려되지 않을 뿐입니다.
 
## 🤔 생성하고, 저장하고, 사용한다...?

변수 활용에서 당연하게 생각했던 것에 대해 깊게 생각해보신 적이 있나요??
- 변수 생성
- 값 저장
- 값 불러오기(변수 사용)

아래는 간단한 C/C++ 코드입니다.
```c++
int a = 1
printf(a)
```

위의 코드를 실행하면, 1이 출력됩니다. 너무 당연한 말이죠??
그런데 생각보다 쉽지 않습니다. 
python은 어떻게 `a`가 1이라는 것을 알까요?

위의 코드가 정상 동작하기 위해서 아래의 항목이 만족되어야 합니다.
- 어딘가에 1이라는 값이 저장되어 있다.
- `a`가 어딘가와 연결되어 있다.

`print(a)`가 실행되는 순간, 
1. python은 `a`와 연결된 어딘가를 봅니다. 
2. 어딘가에 저장되어 있는 값을 읽습니다.
3. 그 곳에서 읽은 1의 값을 `print()`에 전달합니다.
4. `1` 출력!

## 📝 어딘가가 어딘데?

위의 프로세스에서 알 수 있듯이, `a`에 1이 저장되는 것은 아닙니다. 그저 그렇게 보일 뿐입니다.
실제로 데이터가 저장되는 곳은 '어딘가'라는 것이죠!

보통 PC를 구매할 때, 고려하는 3가지가 있죠.
- CPU
- 그래픽 카드
- 메모리

실제로 값이 저장되는 공간이 바로 '메모리'입니다!

메모리는 0, 1을 기록할 수 있는 메모장입니다. 우리가 눈으로 보기 힘들 정도로 많은 0과 1이 있죠.

`a = 1`을 실행하는 순간, 메모리의 어느 곳에다가 1이라는 값을 기록합니다. 
그리고 값을 기록한 위치인 메모리의 주소를 `a`변수와 매칭시킵니다.
그럼 `a`를 사용할 때마다 매칭된 메모리의 주소에 저장된 값을 사용하는 것입니다.

메모리는 지웠다가 다시 쓸 수 있는 연습장 같은 녀석입니다.
따라서, 메모리 공간이 크면 클수록 한번에 많은 값을 저장/활용할 수 있겠죠!

## 💾 변수의 종류...??

C/C++를 사용해보신 분들이라면 머리 속에 떠오르는 단어가 몇 개 있으실 겁니다.

- char
- int
- long long
- ...

python을 사용하신다면, `이게 뭔가..` 싶을 겁니다.
`a`의 값을 불러온다고 생각을 해보죠. 
python은 `a`에 매칭된 메모리 주소로 이동합니다.
값을 읽어볼까..??

어디까지 읽어야할지 난감합니다. 어디까지 읽어야할지 확실하게 정하는 것이 좋겠죠.
이런 이유와 메모리 효율화를 위해 변수의 종류가 생기게 됩니다.

`a`는 `int`로 생성하였기 때문에 4-byte만큼만 읽어서 값을 사용하게 됩니다.

python은 변수 type에 대해 고민하지 않아도 됩니다.
개발자가 고민할 것 없이 python 자체적으로 해결합니다. 덕분에 사용은 편하지만, 세부 동작을 따지기에는 어려움이 있죠.
